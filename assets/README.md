# 资产目录说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`assets` 用于存放项目通用资产，例如图标、样例数据、导出素材和发布资源。

## 2. 背景与问题

通用资产如果散落在各章节，会影响复用和发布整理。

## 3. 核心概念

资产包括但不限于图标、Logo、样例数据、导出封面和公共附件。

## 4. 应用架构

通用资产放入 `assets`，章节专属图片放入 `images`。

## 5. 工作流程

新增资产时记录用途、来源和适用范围。

## 6. 企业案例

企业 AI 平台 Demo 的样例数据可以放入本目录。

## 7. 技术实现建议

敏感数据和真实客户资料不得直接放入本目录。

## 8. 常见问题

问：案例附件应该放 assets 还是 cases？  
答：案例专属附件放 `cases`，跨章节通用资产放 `assets`。

## 9. 后续延伸

补充资产清单和授权记录。
