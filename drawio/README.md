# Draw.io目录说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`drawio` 用于保存复杂架构图、部署图和业务流程图的 Draw.io 源文件。

## 2. 背景与问题

复杂图形需要源文件维护，方便后续迭代和导出。

## 3. 核心概念

Draw.io 适合绘制多层架构、系统集成、部署拓扑和复杂业务流程。

## 4. 应用架构

本目录与 `docs` 章节目录保持对应。

## 5. 工作流程

创建源文件，导出图片到 `images`，在 Markdown 中引用图片并说明源文件位置。

## 6. 企业案例

企业 AI 平台总体架构图可使用 Draw.io 绘制。

## 7. 技术实现建议

每个导出图片都应能找到对应 Draw.io 源文件。

## 8. 常见问题

问：Draw.io 文件是否需要提交 Git？  
答：需要，图形源文件是项目资产的一部分。

## 9. 后续延伸

补充 Draw.io 命名规范和导出约定。
