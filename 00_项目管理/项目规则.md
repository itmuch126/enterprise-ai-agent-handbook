# 项目规则

版本：v1.4  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 0. 项目名称与范围

项目统一名称：《企业AI平台与Agent应用落地手册》。

本项目覆盖企业 AI 平台、AI 知识库、RAG、Agent、MCP、Workflow、Multi-Agent、A2A、企业业务场景、私有化部署、权限安全、治理运营、可观测性和持续交付。

## 1. 基础文档规则

1. 所有 Markdown 文件统一使用 UTF-8 编码。
2. 文档文件名使用中文，目录名前缀使用两位数字排序。
3. 每篇文档必须包含：版本号、更新时间、适用对象、核心结论。
4. 每篇核心文章尽量包含：背景、概念、架构图、流程图、企业案例、落地建议、常见问题。
5. 图片文件统一放入 images 对应章节目录。
6. Mermaid 源文件统一放入 mermaid 对应章节目录。
7. Draw.io 源文件统一放入 drawio 对应章节目录。
8. 示例代码统一放入 examples。
9. 企业案例统一放入 cases。
10. 后续所有内容更新后，需要同步更新 SUMMARY.md。
11. 暂不接入飞书 CLI，飞书仅作为后续手工复制或发布展示层。
12. 本项目优先服务于企业 AI 平台、AI 知识库、RAG、Agent、MCP、Workflow、Multi-Agent、HR Agent、薪酬 Agent、Office Agent、ERP Agent、私有化部署与治理运营等方向。

## 2. Agent流程编排、规则引擎与性能设计约束

1. 在企业 AI 平台手册和企业 AI 知识库后续所有架构设计、Agent 设计、流程编排设计中，必须考虑 Agent 流程编排过程中引入规则引擎的可能性。
2. 规则引擎用于处理确定性判断、业务规则匹配、流程分支控制、权限判断、数据校验、路由决策等场景。
3. 不能把所有判断都交给大模型。确定性规则应优先由规则引擎、配置中心或业务服务处理，大模型主要负责自然语言理解、上下文归纳、非确定性推理和辅助生成。
4. Agent 编排设计必须考虑性能问题，包括：
   - 规则执行效率；
   - Agent 多步骤调用耗时；
   - 大模型调用延迟；
   - 并发请求处理能力；
   - 缓存设计；
   - 异步任务与消息队列；
   - 超时控制；
   - 降级与兜底策略；
   - 执行链路追踪；
   - Token 消耗控制。
5. 后续文档中凡涉及 Agent、智能体、流程编排、知识库问答、自动化业务处理时，都需要单独说明规则引擎与性能设计。
6. 架构设计中应明确哪些判断属于确定性规则、哪些判断可由大模型辅助、哪些判断必须进入人工确认或审批流程。
7. 涉及企业核心数据、权限、安全、薪酬、财务、ERP 写操作等高风险场景时，规则引擎、业务服务和审批流程优先级高于大模型输出。

## 3. 组织人员、菜单权限与审计设计约束

1. 后续所有 Agent、RAG、Workflow、MCP、业务系统集成设计，都必须考虑组织、人员、角色、菜单、权限、数据权限和审计留痕。
2. 组织、部门、人员、账号、岗位、角色、菜单、权限是企业 AI 平台基础主数据，应作为所有 Agent、RAG、Workflow、MCP 和业务系统集成的统一底座。
3. 用户访问 Agent 前必须进行身份认证和权限校验；Agent 查询知识库、调用工具、访问业务系统、执行操作时，必须进行二次权限判断。
4. 权限控制至少包括菜单权限、功能权限、数据权限、接口权限、工具调用权限、Agent 使用权限和审批权限。
5. 数据权限需要结合组织、部门、岗位、角色、人员范围、业务对象和敏感级别进行控制。
6. 涉及薪酬、财务、合同、订单、结算、发票、用户隐私等敏感数据时，必须考虑脱敏、最小权限、用户确认和审计追踪。
7. 权限判断应优先由 IAM、权限中心、规则引擎或业务系统处理，不能依赖大模型自行判断。
8. 大模型主要负责语义理解、内容生成和辅助分析，不负责最终权限裁决。
9. 所有权限判断结果、拒绝原因、高风险操作确认和敏感数据访问都需要记录到审计日志中。

## 4. 统一画图规范

1. 后续所有重要章节必须按照统一画图规范设计图示。
2. 不同类型图回答不同问题，不允许只用一张总架构图替代所有图。
3. Agent、RAG、Workflow、MCP、Multi-Agent、业务 Agent、权限、安全、部署、数据治理等章节，应至少包含静态架构图、动态流程图、时序图或部署图中的若干种。
4. Mermaid 源文件必须保存到 `mermaid` 目录。
5. 图示需要服务于汇报、架构设计、开发实现、运维实施和业务理解。
6. 高风险业务流程图必须体现权限校验、规则引擎、人工确认、审批和审计留痕。
