# MCP协议基础

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

MCP 为 Agent 连接外部工具和数据源提供统一协议思路，企业可用它降低工具接入成本和治理复杂度。

## 2. 背景与问题

没有统一协议时，每个 Agent 都要为每个系统单独适配工具调用，导致复用困难和权限治理分散。

## 3. 核心概念

- MCP Server：对外暴露工具、资源和提示能力的服务。
- MCP Client：连接 MCP Server 并发起调用的客户端。
- Tool：可被模型或 Agent 调用的结构化能力。

## 4. 应用架构

企业可在 Agent 平台和业务系统之间加入 MCP 服务层，统一封装系统能力。

## 5. 工作流程

Agent 发现工具，读取工具描述，生成调用参数，MCP Server 执行业务动作并返回结果。

## 6. 企业案例

HR MCP Server 可封装员工查询、组织架构查询、制度文档检索和流程发起能力。

## 7. 技术实现建议

优先把只读查询能力 MCP 化，再逐步开放写操作和审批触发能力。

## 8. 常见问题

问：MCP 是否替代企业 API 网关？  
答：不替代。MCP 更偏 Agent 工具协议，API 网关仍负责传统系统接入和流量治理。
