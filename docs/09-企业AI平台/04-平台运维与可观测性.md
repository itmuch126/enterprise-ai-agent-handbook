# 平台运维与可观测性

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

AI 平台上线后必须可观测，包括模型调用、知识检索、工具执行、成本、延迟、错误和用户反馈。

## 2. 背景与问题

AI 应用的不稳定性来自模型、知识、工具和业务系统多个环节，传统接口监控不足以定位问题。

## 3. 核心概念

- Trace：一次 Agent 任务的全链路记录。
- Metric：延迟、成本、成功率、命中率等指标。
- Feedback：用户对结果的确认、纠错和评价。

## 4. 应用架构

可观测系统应覆盖前端会话、后端任务、模型网关、RAG 链路和工具调用。

## 5. 工作流程

记录请求，追踪子步骤，汇总指标，发现异常，回放问题样例，优化平台能力。

## 6. 企业案例

当 HR Agent 回答错误时，应能回溯使用了哪个文档片段、哪个模型版本和哪个提示词版本。

## 7. 技术实现建议

为每次 Agent 执行生成 traceId，并贯穿模型、RAG 和工具调用日志。

## 8. 常见问题

问：是否只需要记录最终回答？  
答：不够。必须记录关键中间步骤，否则无法定位问题来源。

## 9. 后续延伸

补充日志字段、指标看板和告警策略。
