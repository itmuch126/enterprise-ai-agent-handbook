# Workflow基础

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

Workflow 适合承载确定性强、需要审批和状态追踪的业务流程，是企业 Agent 可靠落地的重要补充。

## 2. 背景与问题

Agent 擅长理解和编排，但企业流程通常需要明确节点、责任人、状态、权限和审计记录。

## 3. 核心概念

- 节点：流程中的一个处理步骤。
- 状态：流程实例当前所处阶段。
- 编排：把多个任务、工具和人工审批组合成完整流程。

## 4. 应用架构

Workflow 引擎可与 Agent Runtime 协同，由 Agent 理解任务并触发确定性流程。

## 5. 工作流程

用户提出目标，Agent 识别流程类型，填充参数，触发 Workflow，等待或汇总执行结果。

## 6. 企业案例

员工申请证明材料时，HR Agent 收集信息并触发 OA 审批流程。

## 7. 技术实现建议

关键业务写操作应优先进入 Workflow，而不是由 Agent 直接执行。

## 8. 常见问题

问：Workflow 和 Agent 是否冲突？  
答：不冲突。Workflow 提供确定性执行，Agent 提供自然语言理解和动态编排。
