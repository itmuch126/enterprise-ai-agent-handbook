# 业务场景建模方法

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

业务场景建模要先明确用户、目标、输入、输出、知识、工具、权限和异常路径。

## 2. 背景与问题

没有场景建模的 Agent 容易变成泛化聊天入口，难以验收，也难以长期运营。

## 3. 核心概念

- 用户旅程：用户完成任务的关键步骤。
- 业务对象：流程中涉及的数据实体。
- 工具动作：Agent 可执行的系统操作。
- 验收指标：效果是否达标的判断标准。

## 4. 应用架构

场景模型应映射到知识库、工具服务、Workflow 和权限策略。

## 5. 工作流程

访谈业务，整理任务清单，拆解流程，定义工具，设计异常处理，建立评估集。

## 6. 企业案例

招聘 Agent 的场景模型应覆盖职位发布、简历筛选、面试安排和候选人沟通。

## 7. 技术实现建议

为每个 Agent 建立一页场景卡，作为需求、设计、开发和验收的共同依据。

## 8. 常见问题

问：能否先开发再补场景模型？  
答：不建议。场景边界不清会导致开发返工和验收困难。
