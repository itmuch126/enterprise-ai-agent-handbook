# Spring Boot Agent服务

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

Spring Boot 适合承载企业 Agent 的 API、权限、工具适配、审计日志和平台管理能力。

## 2. 背景与问题

企业后端需要与用户体系、组织权限、业务系统和数据库集成，Java 技术栈在这类场景中更稳定。

## 3. 核心概念

- AgentController：对外提供对话和任务接口。
- AgentService：封装任务编排和模型调用。
- ToolAdapter：适配企业业务系统。
- AuditLog：记录任务和工具调用。

## 4. 应用架构

Spring Boot 服务通过模型网关调用 LLM，通过 MCP 或内部 API 调用工具，并写入数据库。

## 5. 工作流程

接收请求，校验权限，创建任务，调用模型和工具，保存日志，返回结果。

## 6. 企业案例

HR Agent 后端提供政策问答、员工查询和流程发起接口。

## 7. 技术实现建议

工具调用必须通过服务层封装，避免模型直接接触数据库或内部接口细节。

## 8. 常见问题

问：模型 API Key 可以放在前端吗？  
答：不可以。必须保存在后端或密钥管理系统中。
