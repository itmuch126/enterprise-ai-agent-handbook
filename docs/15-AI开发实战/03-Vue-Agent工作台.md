# Vue Agent工作台

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

Vue Agent 工作台不应只是聊天界面，还应包含任务状态、工具调用、引用来源、审批确认和反馈入口。

## 2. 背景与问题

企业用户需要看到 Agent 的执行过程和结果来源，否则难以信任 AI 输出。

## 3. 核心概念

- 会话面板：承载用户输入和 Agent 回复。
- 任务轨迹：展示执行步骤和状态。
- 引用来源：展示 RAG 文档来源。
- 人工确认：执行写操作前的确认界面。

## 4. 应用架构

前端通过 REST 或 SSE/WebSocket 与后端交互，展示流式响应和任务状态。

## 5. 工作流程

用户选择 Agent，输入任务，查看执行过程，确认关键操作，提交反馈。

## 6. 企业案例

薪酬 Agent 在展示薪资解释时，应同时展示引用规则和敏感数据提示。

## 7. 技术实现建议

为不同 Agent 提供差异化任务面板，而不是所有场景共用一个聊天框。

## 8. 常见问题

问：是否必须支持流式输出？  
答：不是必须，但流式输出能改善长任务的等待体验。
