# 开发环境与工程结构

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

企业级 AI Agent Demo 建议采用 Spring Boot 后端、Vue 前端和 Python AI 服务的分层工程结构。

## 2. 背景与问题

AI Demo 容易只关注模型调用，忽略企业工程需要的权限、配置、日志、测试和部署。

## 3. 核心概念

- Spring Boot：承载企业业务 API、权限和 Agent 服务。
- Vue：承载 Agent 工作台和管理界面。
- Python：承载 RAG、Embedding、评估和实验能力。

## 4. 应用架构

工程可划分为 `examples/springboot`、`examples/vue`、`examples/python`、`examples/mcp`、`examples/rag` 和 `examples/agent`。

## 5. 工作流程

先建立接口契约，再实现后端服务和前端页面，最后接入模型、RAG 和工具调用。

## 6. 企业案例

以 HR Agent 作为第一条端到端链路，验证知识问答、工具调用和审计日志。

## 7. 技术实现建议

先创建最小可运行 Demo，再逐步增加权限、评估和运维能力。

## 8. 常见问题

问：是否必须同时开发三套工程？  
答：不必须。可以先用 Spring Boot 完成主链路，再拆分 Python AI 服务。
