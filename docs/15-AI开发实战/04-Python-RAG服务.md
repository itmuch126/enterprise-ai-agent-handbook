# Python RAG服务

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

Python 服务适合承载 RAG、Embedding、文档解析、评估和模型实验能力，并通过 API 被企业后端调用。

## 2. 背景与问题

RAG 相关生态在 Python 中更丰富，但企业主系统常由 Java 承载，需要清晰的服务边界。

## 3. 核心概念

- Ingestion：文档入库流程。
- Retrieval：知识检索流程。
- Evaluation：RAG 效果评估。
- API Contract：与 Java 后端的接口契约。

## 4. 应用架构

Python RAG 服务提供文档导入、向量检索、重排和评估接口，Java 后端负责权限和业务编排。

## 5. 工作流程

导入文档，生成索引，接收查询，返回相关片段，后端组合提示词并调用模型。

## 6. 企业案例

HR 知识库由 Python 服务完成文档解析和检索，Spring Boot 负责用户权限和响应生成。

## 7. 技术实现建议

Python 服务不应绕过企业权限系统直接返回敏感知识。

## 8. 常见问题

问：RAG 服务是否一定要独立部署？  
答：不一定。早期可嵌入后端，复杂后再拆分。

## 9. 后续延伸

补充 RAG API、向量库选型和评估脚本。
