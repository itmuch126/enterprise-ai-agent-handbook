# A2A协议与协作

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

A2A 关注 Agent 之间的互操作和任务交接，是多智能体体系从内部实现走向标准协作的关键方向。

## 2. 背景与问题

企业内部和外部可能存在多个 Agent 平台，如果缺少协作协议，任务交接会依赖定制接口。

## 3. 核心概念

- Agent Card：描述 Agent 能力、输入输出和调用方式。
- Task Handoff：跨 Agent 传递任务和上下文。
- Capability Discovery：发现可用 Agent 能力。

## 4. 应用架构

A2A 可位于 Agent 平台之间，提供身份认证、能力发现、任务传递和状态同步能力。

## 5. 工作流程

发起方发现目标 Agent，传递任务描述和必要上下文，目标 Agent 执行后返回状态和结果。

## 6. 企业案例

企业内部 HR Agent 可将合同审查任务交给法务 Agent，并接收结构化审查意见。

## 7. 技术实现建议

跨 Agent 协作必须明确身份、权限、数据范围、超时和失败回退。

## 8. 常见问题

问：A2A 是否只适合跨公司协作？  
答：不是。企业内部多平台、多团队 Agent 协作同样需要协议化。
