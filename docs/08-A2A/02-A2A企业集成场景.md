# A2A企业集成场景

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

A2A 更适合跨域、跨团队、跨平台的复杂协作场景，尤其适合企业生态系统集成。

## 2. 背景与问题

不同系统和供应商可能各自提供 Agent 能力，企业需要统一管理它们之间的任务流转。

## 3. 核心概念

- 内部 A2A：企业内部不同业务域 Agent 协作。
- 外部 A2A：企业与供应商、客户或平台 Agent 协作。
- 协作边界：定义数据、动作和责任范围。

## 4. 应用架构

企业可建设 A2A Gateway，统一处理 Agent 身份认证、能力注册、任务路由和调用审计。

## 5. 工作流程

注册外部 Agent，审核能力范围，配置调用策略，执行任务交接，记录全过程。

## 6. 企业案例

采购 Agent 可与供应商报价 Agent 协作，收集报价后交给 ERP Agent 生成采购建议。

## 7. 技术实现建议

外部 Agent 接入必须经过安全评估，敏感数据默认不出域。

## 8. 常见问题

问：是否应该一开始就建设 A2A Gateway？  
答：不建议。应在已有多个稳定 Agent 后再抽象平台级能力。
