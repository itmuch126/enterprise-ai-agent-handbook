# RAG评估与优化

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

RAG 优化要分别评估检索、重排、上下文构造和答案生成，不能只看最终回答是否“像是正确”。

## 2. 背景与问题

RAG 效果差可能来自文档质量、切分策略、向量模型、召回参数、重排模型或提示词约束。

## 3. 核心概念

- Recall：相关片段是否被召回。
- Precision：召回片段是否足够相关。
- Faithfulness：回答是否忠于引用内容。
- Citation：回答是否能给出来源。

## 4. 应用架构

评估系统应保存问题、标准答案、召回片段、模型回答、人工评分和版本信息。

## 5. 工作流程

构建评估集，批量运行 RAG，分析失败样例，调整切分和检索参数，再做回归测试。

## 6. 企业案例

HR 政策问答评估集应覆盖假勤、福利、入转调离、组织和权限边界问题。

## 7. 技术实现建议

每次知识库更新后执行核心问题回归，防止新文档影响已有高频问题。

## 8. 常见问题

问：RAG 优化是否优先换更强模型？  
答：不一定。很多问题来自知识加工和检索链路，先定位瓶颈更重要。
