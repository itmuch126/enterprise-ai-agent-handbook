# RAG基础架构

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

RAG 通过检索企业知识增强模型回答，是企业知识库和 Agent 问答能力的基础架构。

## 2. 背景与问题

模型不天然掌握企业内部制度、流程和业务数据，直接回答容易产生过时或错误内容。

## 3. 核心概念

- Chunk：文档切分后的知识片段。
- Embedding：文本向量表示。
- Retriever：负责召回相关知识。
- Rerank：对召回结果重新排序。

## 4. 应用架构

RAG 通常包含文档采集、清洗切分、向量化、索引、检索、重排、上下文拼接和答案生成。

## 5. 工作流程

导入文档，切分片段，生成向量，建立索引，用户提问，检索相关内容，模型生成带引用答案。

## 6. 企业案例

HR 制度问答可基于员工手册、假勤制度、福利政策和流程说明构建 RAG 知识库。

## 7. 技术实现建议

必须保留来源引用、版本信息和更新时间，避免知识库回答无法追溯。

## 8. 常见问题

问：RAG 是否能解决所有幻觉问题？  
答：不能。RAG 能降低风险，但还需要检索质量、提示词约束和结果校验。
