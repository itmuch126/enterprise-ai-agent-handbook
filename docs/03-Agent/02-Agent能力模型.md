# Agent能力模型

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

企业 Agent 能力应拆分为感知、理解、规划、执行、记忆、反馈和治理七类能力。

## 2. 背景与问题

如果只把 Agent 看成模型接口，系统会缺少权限控制、状态管理、异常处理和持续优化能力。

## 3. 核心概念

- 感知：接收文本、文件、表格、系统事件等输入。
- 规划：拆解任务并选择执行路径。
- 执行：调用工具、API、Workflow 或 MCP 服务。
- 治理：权限、审计、成本、质量和安全控制。

## 4. 应用架构

能力模型可以映射到平台模块，包括 Agent Runtime、Tool Registry、Memory Store、Policy Engine 和 Observability。

## 5. 工作流程

能力建模应先明确业务边界，再定义工具集合、数据权限和失败回退策略。

## 6. 企业案例

薪酬 Agent 应具备制度解释、数据查询、权限判断和敏感信息脱敏能力。

## 7. 技术实现建议

为每类能力定义接口和指标，避免不同 Agent 之间重复实现基础能力。

## 8. 常见问题

问：是否每个 Agent 都需要完整七类能力？  
答：不一定，应按业务风险和任务复杂度裁剪。
