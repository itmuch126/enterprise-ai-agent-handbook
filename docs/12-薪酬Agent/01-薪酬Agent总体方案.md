# 薪酬Agent总体方案

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

薪酬 Agent 是高价值但高敏感场景，必须优先设计权限、脱敏、审计和人工确认机制。

## 2. 背景与问题

薪酬规则复杂、咨询频繁、数据敏感，普通问答系统难以同时满足准确性和安全性。

## 3. 核心概念

- 薪酬政策问答：解释薪酬制度和发薪规则。
- 薪资单解释：帮助员工理解本人薪资构成。
- 计薪辅助：帮助薪酬专员检查异常和规则匹配。

## 4. 应用架构

薪酬 Agent 接入薪酬知识库、薪资系统、员工权限、审计日志和敏感信息脱敏服务。

## 5. 工作流程

用户提问，系统校验身份和权限，检索知识或查询数据，生成解释并记录审计。

## 6. 企业案例

员工询问本月实发工资变化时，Agent 只展示本人可见项目，并引用对应薪酬规则。

## 7. 技术实现建议

默认只开放本人数据查询，批量薪酬分析必须限制给授权角色。

## 8. 常见问题

问：薪酬 Agent 能否直接修改薪酬数据？  
答：不建议直接修改，写操作应进入审批或薪酬系统流程。

## 9. 后续延伸

补充薪酬 Agent 权限矩阵和审计字段。
