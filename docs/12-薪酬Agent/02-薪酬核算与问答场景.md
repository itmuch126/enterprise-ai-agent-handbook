# 薪酬核算与问答场景

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

薪酬核算适合作为专员辅助场景，员工问答适合作为服务入口，两者权限和输出边界必须区分。

## 2. 背景与问题

员工关注薪资解释，薪酬专员关注核算效率、异常识别和规则一致性。

## 3. 核心概念

- 员工问答：本人薪资单解释、发薪时间、规则说明。
- 专员辅助：异常检测、规则匹配、批量核对。
- 敏感字段：薪资、奖金、个税、社保、公积金。

## 4. 应用架构

按角色分为员工端、专员端和管理员端，并分别配置数据权限和操作权限。

## 5. 工作流程

员工侧以问答为主，专员侧以任务清单、异常报告和规则校验为主。

## 6. 企业案例

薪酬专员上传核算结果后，Agent 标记异常波动并提示可能原因。

## 7. 技术实现建议

敏感输出应进行脱敏、分级授权和全量审计。

## 8. 常见问题

问：员工能否查看同部门薪资均值？  
答：取决于企业制度和权限策略，默认不开放敏感聚合数据。
