# 参考资料

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

参考资料用于沉淀外部标准、官方文档、论文、工程实践和内部资料来源，便于后续查证。

## 2. 背景与问题

AI 技术变化快，文档中的判断需要保留来源和更新时间。

## 3. 核心概念

- 官方文档：优先引用协议、平台和框架的原始资料。
- 工程实践：记录可复用的设计经验。
- 内部资料：记录企业内部业务制度和系统说明。

## 4. 应用架构

参考资料应按技术主题和业务主题分类管理。

## 5. 工作流程

新增资料时记录标题、来源、链接、访问时间和适用章节。

## 6. 企业案例

薪酬 Agent 的制度依据应引用企业最新薪酬政策文档。

## 7. 技术实现建议

不要把未经验证的二手资料作为架构决策依据。

## 8. 常见问题

问：参考资料是否必须全部公开可访问？  
答：不必须，但内部资料应标明权限范围。

## 9. 后续延伸

补充参考资料清单和引用格式。
