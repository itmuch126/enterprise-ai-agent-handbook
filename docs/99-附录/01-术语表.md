# 术语表

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

统一术语是企业 AI 平台建设的重要基础，可以减少业务、产品、研发和管理层之间的沟通偏差。

## 2. 背景与问题

AI Agent、RAG、MCP、Workflow、A2A 等概念容易被混用，需要在项目内保持一致定义。

## 3. 核心概念

- LLM：大语言模型。
- RAG：检索增强生成。
- Agent：面向目标执行任务的智能体。
- MCP：模型上下文协议相关工具接入方式。
- Workflow：确定性业务流程编排。
- A2A：Agent 与 Agent 之间的协作方式。

## 4. 应用架构

术语表作为所有文档的基础引用，后续章节新增概念时应同步更新。

## 5. 工作流程

发现新术语，补充定义、适用范围和相关章节链接。

## 6. 企业案例

在 HR Agent 方案中，应区分知识问答、流程助手和自动化执行。

## 7. 技术实现建议

术语定义要短、明确，并避免与供应商营销概念混淆。

## 8. 常见问题

问：术语表是否需要版本管理？  
答：需要，术语变化会影响整套文档理解。
