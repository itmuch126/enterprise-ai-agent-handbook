# 模型选型与评估

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

企业模型选型应以场景评估为准，而不是只看榜单排名或单次体验。

## 2. 背景与问题

不同模型在中文理解、长上下文、工具调用、代码能力、成本和响应速度上差异明显。

## 3. 核心概念

- 离线评估：基于标准样例集批量测试。
- 在线评估：在真实业务流量中观察效果。
- 成本评估：综合 Token 费用、延迟、并发和运维成本。

## 4. 应用架构

建议通过模型网关屏蔽供应商差异，并支持多模型路由和灰度切换。

## 5. 工作流程

定义业务样例，执行模型对比，人工标注结果，汇总指标，进入灰度验证。

## 6. 企业案例

Office Agent 可能更看重文档生成质量，ERP Agent 则更看重结构化工具调用稳定性。

## 7. 技术实现建议

每个 Agent 都应有独立评估集，避免用通用问答结果推断业务场景效果。

## 8. 常见问题

问：是否只需要选择一个模型？  
答：通常不够。企业平台应支持按任务路由到不同模型。

## 9. 后续延伸
