# HR-Agent总体方案

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

HR Agent 应从员工服务和 HR 运营提效切入，逐步扩展到招聘、入转调离和组织管理。

## 2. 背景与问题

HR 团队面对大量重复咨询、流程指引和政策解释工作，适合通过 Agent 提升响应效率。

## 3. 核心概念

- 员工服务：政策问答、证明办理、流程查询。
- HR 运营：入职清单、组织查询、数据统计。
- 权限边界：员工、主管、HRBP、管理员权限不同。

## 4. 应用架构

HR Agent 接入 HR 知识库、组织人员系统、OA 流程和通知渠道。

## 5. 工作流程

用户提问或发起任务，Agent 判断场景，检索制度或调用工具，输出答案或触发流程。

## 6. 企业案例

员工询问年假规则时，Agent 根据员工入职时间、制度文档和适用地区给出解释。

## 7. 技术实现建议

先实现政策问答和流程指引，再接入人员数据查询和流程发起。

## 8. 常见问题

问：HR Agent 能否直接给员工做劳动关系判断？  
答：高风险判断应交由 HR 专员确认，Agent 只提供辅助信息。
