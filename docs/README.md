# 文档目录说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`docs` 是项目核心文档目录，按章节编号组织企业 AI Agent 架构设计与实践内容。

## 2. 背景与问题

文档需要按主题长期演进，因此必须保持稳定目录和清晰命名。

## 3. 核心概念

章节目录使用两位数字前缀排序，文件名使用中文表达主题。

## 4. 应用架构

文档章节与 `images`、`mermaid`、`drawio` 下的同名目录对应。

## 5. 工作流程

新增文章后，需要同步更新根目录 `SUMMARY.md`。

## 6. 企业案例

业务案例正文可放在对应章节，完整案例材料放入 `cases`。

## 7. 技术实现建议

核心文章应包含背景、概念、架构图、流程图、企业案例和落地建议。

## 8. 常见问题

问：是否可以跳过章节编号？  
答：可以，但需要保持目录排序和 `SUMMARY.md` 一致。

## 9. 后续延伸

后续可补充自动目录生成脚本。
