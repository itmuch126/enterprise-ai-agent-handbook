# 工具目录说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`tools` 用于存放后续辅助脚本和工具，例如目录生成、文档检查、图形导出和发布转换。

## 2. 背景与问题

文档规模增长后，需要工具保证目录、链接、模板和发布格式一致。

## 3. 核心概念

工具脚本应围绕 Markdown 检查、SUMMARY 同步、图片校验和发布转换展开。

## 4. 应用架构

工具应以本地 Markdown 为唯一数据源，不直接修改发布端内容。

## 5. 工作流程

先人工维护文档结构，再逐步沉淀自动化检查和生成脚本。

## 6. 企业案例

可编写脚本检查每篇文章是否包含版本号、更新时间、适用对象和核心结论。

## 7. 技术实现建议

工具脚本应可重复执行，并避免覆盖人工编辑内容。

## 8. 常见问题

问：是否现在接入飞书 CLI？  
答：暂不接入，飞书仅作为后续展示和发布层。

## 9. 后续延伸

补充 Markdown 质量检查脚本。
