# Mermaid目录说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`mermaid` 用于保存流程图、架构图和时序图的 Mermaid 源文件。

## 2. 背景与问题

图形源文件必须可维护，不能只保留导出的图片。

## 3. 核心概念

Mermaid 适合绘制流程图、序列图、状态图和简单架构图。

## 4. 应用架构

本目录与 `docs` 章节目录保持对应。

## 5. 工作流程

新增图形时先创建 Mermaid 源文件，再按需要导出图片到 `images`。

## 6. 企业案例

Agent 工具调用流程图可放入 `mermaid/03-Agent`。

## 7. 技术实现建议

图形文件名应与引用它的文档主题保持一致。

## 8. 常见问题

问：复杂架构图是否一定用 Mermaid？  
答：不一定，复杂图可使用 Draw.io。

## 9. 后续延伸

补充 Mermaid 图形模板。
