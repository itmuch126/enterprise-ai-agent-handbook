# 图片目录说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`images` 用于存放文档引用的图片文件，并按章节目录归档。

## 2. 背景与问题

图片集中管理可以避免文档迁移或发布时资源丢失。

## 3. 核心概念

图片文件应尽量使用清晰命名，能反映章节和用途。

## 4. 应用架构

本目录与 `docs` 章节目录保持对应。

## 5. 工作流程

新增图片时放入对应章节目录，并在 Markdown 中使用相对路径引用。

## 6. 企业案例

企业 AI 平台总体架构图可放入 `images/09-企业AI平台`。

## 7. 技术实现建议

优先保留源文件在 `mermaid` 或 `drawio`，图片只作为渲染结果。

## 8. 常见问题

问：截图是否可以直接放根目录？  
答：不建议，应按章节归档。

## 9. 后续延伸

补充图片命名规范。
