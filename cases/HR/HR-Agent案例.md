# HR-Agent案例

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 案例结论

HR Agent 适合从政策问答、流程指引和员工服务切入，逐步扩展到招聘和组织运营。

## 2. 业务场景

员工经常咨询假勤、福利、入职、转岗、离职和证明办理等问题。

## 3. 关键角色

员工、直属主管、HRBP、HR 共享服务中心、系统管理员。

## 4. 当前痛点

咨询重复、制度分散、流程路径复杂、HR 响应压力大。

## 5. AI Agent 方案

通过 HR 知识库、员工数据查询和 OA 流程工具构建员工服务 Agent。

## 6. 系统架构

HR Agent 接入企业 AI 平台、HR 系统、OA 系统、知识库和审计日志。

## 7. 工作流程

员工提问后，Agent 校验权限，检索制度，必要时查询人员信息或发起流程。

## 8. 落地路径

先做制度问答，再做流程指引，最后接入员工数据和流程发起。

## 9. 效果评估

关注咨询解决率、HR 人工转接率、员工满意度和错误回答率。

## 10. 风险与注意事项

劳动关系、薪酬、绩效等敏感问题必须保留人工复核。
