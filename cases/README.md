# 案例库说明

版本：v1.0  
更新时间：2026-06-29  
适用对象：企业软件工程师 / 架构师 / 技术负责人  

## 1. 本章核心结论

`cases` 用于沉淀企业 AI Agent 在不同业务域的案例材料、需求分析和落地方案。

## 2. 背景与问题

企业 AI 落地必须结合真实业务场景，不能只停留在通用技术说明。

## 3. 核心概念

案例目录按业务域划分，包括 HR、Payroll、ERP、OA、Office、Manufacturing 和 Finance。

## 4. 应用架构

案例材料应能反向支撑章节正文、架构图、流程图和 Demo 需求。

## 5. 工作流程

记录业务背景、角色、痛点、Agent 方案、系统架构、流程和风险。

## 6. 企业案例

首批案例围绕 HR Agent、薪酬 Agent、ERP Agent、OA Agent 和 Office Agent 展开。

## 7. 技术实现建议

每个案例都应使用 `templates/案例分析模板.md` 作为起点。

## 8. 常见问题

问：案例是否必须包含真实客户名称？  
答：不需要，可以使用脱敏或抽象后的企业场景。

## 9. 后续延伸

补充案例评分矩阵和优先级列表。
